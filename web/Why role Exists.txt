1️⃣ Why role Exists in user.model.js

In your schema we defined:

role: {
  type: String,
  enum: ["admin", "operator"],
  default: "operator"
}

Meaning:

Role	Purpose
admin	full system access
operator	limited access

This is called Role-Based Access Control (RBAC).

2️⃣ Why We Didn't Send Role During Register

Because we set:

default: "operator"

So when a user registers:

{
  "name": "Falgun",
  "email": "falgun@test.com",
  "password": "123456"
}

MongoDB automatically stores:

{
  "name": "Falgun",
  "email": "falgun@test.com",
  "password": "...hashed...",
  "role": "operator"
}

So role is automatically assigned.

3️⃣ Why Role Is Important for Your Hackathon Project

Your system has different responsibilities.

Example:

Admin

Can:

create inverter
delete inverter
view all users
manage system
Operator

Can:

view dashboard
view predictions
view telemetry

So we restrict APIs.

4️⃣ Example Role Protection Middleware

Create:

middleware/role.middleware.js
const authorizeRoles = (...roles) => {
  return (req, res, next) => {

    if (!roles.includes(req.user.role)) {
      return res.status(403).json({
        message: "Access denied"
      });
    }

    next();
  };
};

export default authorizeRoles;
5️⃣ Example Usage

Example in inverter routes:

router.post(
